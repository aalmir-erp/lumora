Where to find each token & key

Once Meta has verified your business number, follow each step below to gather every value required on the Tokens & Keys page.

A. Meta WhatsApp Cloud API

  1. Open developers.facebook.com/apps → pick your app (e.g. MIR AI Sales BOT).
  2. Sidebar → WhatsApp → API Setup (also called "Try it out" in the new UI).
  3. Phone Number ID: shown next to the active number, top of the page. Copy → paste into Meta Phone Number ID.
  4. WhatsApp Business Account ID: also on the same row. Copy → paste into WhatsApp Business Account ID (optional).
  5. Access Token: click Generate access token. Copy immediately (Meta only shows it once) → paste into Meta Access Token.
    For production, replace the 24-hour temporary token with a permanent System User token: Business Settings → System Users → Add → assign WhatsApp asset → Generate token (full_control + whatsapp_business_messaging + whatsapp_business_management).
  6. Sidebar → App Settings → Basic. The App Secret field has a "Show" button (Meta will ask for your password). Copy → paste into Meta App Secret.
  7. Webhook Verify Token: you invent this. Type any random long string into Meta Webhook Verify Token on the settings page. Then in Meta App → WhatsApp → Configuration → Webhook, paste the SAME string in Meta's "Verify token" box, and set the Callback URL to https://<your-railway-app>/webhook.

B. AI provider key

You only need ONE of these, depending on the AI Provider setting.

  1. Anthropic (recommended for this bot): sign up at console.anthropic.com → Settings → API Keys → "Create key". Copy → paste into Anthropic API Key. Default model claude-sonnet-4-6 works well; leave the model field empty unless you have a reason to override.
  2. OpenAI: sign up at platform.openai.com → API Keys → "Create new secret key". Copy → paste into OpenAI API Key. Default model gpt-4.1.

Make sure you've added a payment method to whichever provider you pick — both reject API calls until billing is enabled.

C. Sales handoff (optional)

  1. Sales handoff WhatsApp number: when the bot can't help (frustration / explicit request / completed order intake), it pings this number with a summary. Use international format with leading +, e.g. +971564020087.
  2. Sales handoff email: shown to customers as a fallback contact.

D. Bot ON / OFF

The Bot enabled? setting takes true or false. Setting it to false tells the bot to drop incoming webhooks silently — useful when you want sales agents to handle everything manually for a while.

The bot also auto-pauses each conversation for 4 hours after escalating, to avoid talking over an agent who's taking the customer over from Meta Inbox.

E. Connecting the webhook to Meta

  1. Make sure your bot is deployed and reachable at https://<your-railway-app>.
  2. Open Meta App → WhatsApp → Configuration → Webhook.
  3. Click Edit on the Webhook row.
  4. Callback URL: https://<your-railway-app>/webhook
  5. Verify token: paste the same string you saved as Meta Webhook Verify Token.
  6. Click Verify and save. Meta calls your bot, your bot replies with the challenge, the connection becomes green.
  7. Below the Webhook row, click Manage on the Webhook fields and subscribe to messages.

You're done. Send a WhatsApp message from your test number → bot replies.
